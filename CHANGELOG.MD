# CHANGELOG

SoonLink / 溯联开源版的主要变更记录。

标记说明：

- `+特性`：新增能力或可见体验升级
- `=修复`：行为修正、稳定性提升、兼容性收口
- `=文档`：README / CHANGELOG / 说明同步
- `=工程`：构建、发布、仓库边界、CI 等工程治理更新

说明：

- `0.1.x` 到 `0.4.x` 用于承接独立公开仓库之前的产品演进摘要，帮助后续版本链保持连续。
- 从 `0.5.x` 开始，SoonLink 进入公开仓库治理阶段，版本、发布和协作流程在仓库内持续留痕。

## [0.5.27] - 2026-04-06

- `=工程` GitHub Release、Docker 与 Homebrew 发布链统一支持 `0.8.27`、`0.0.5.17` 这类纯数字 dotted tag
- `=修复` Release / Docker / Homebrew 工作流增加 tag 与 `cjpm.toml` 版本一致性校验，避免 release 元数据与产物文件名错位
- `=修复` Homebrew CI 改为临时 tap 安装并修正 formula 启动脚本渲染，公开仓自动化验证恢复稳定
- `=文档` README、CHANGELOG、排障文档、GitHub Actions 说明、运行时版本常量与对应测试统一同步到 `0.5.27`

## [0.5.22] - 2026-04-06

- `+特性` 手动节点端点模型从单纯 `ip + port` 升级为 `scheme + host + port + basePath`，Web / CLI / MCP / 自动化运行时统一收口
- `+特性` SoonLink Node 连接器开始支持 `https` 与反向代理子路径，跨节点访问可直接对接带 base path 的部署形态
- `=修复` 手动节点在未显式填写 connector 时，`soonlink-http` 也会严格校验非法 endpoint scheme，避免默认化路径绕过规则
- `=文档` README、CHANGELOG、运行时版本常量、排障文档与 release 草稿统一同步到 `0.5.22`

## [0.5.21] - 2026-04-06

- `+特性` Core 默认产品画像彻底收口到公开能力边界：主动发现、peer、中继、白名单、收藏与增强安全默认不再暴露
- `+特性` Web 控制台默认 edition / capability 上下文切换为 `core`，首页徽标与能力门控和实际公开版保持一致
- `=修复` 占位鉴权 / 加密提供者统一回落到稳定的 `placeholder` 标识，避免 edition 派生名继续泄露预览分支语义
- `=工程` 增加 GitHub Release 制品流水线、跨平台打包脚本和 `linux/amd64 + linux/arm64` Docker 发布默认平台
- `=文档` 版本、GitHub Actions 说明、排障文档与 release 草稿统一同步到 `0.5.21`

## [0.5.17] - 2026-04-04
